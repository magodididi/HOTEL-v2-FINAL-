# Проект "HOTEL-v2-FINAL-"

## Описание проекта
Проект представляет собой систему бронирования отелей, позволяющую пользователям искать отели, бронировать номера, отменять бронирования и управлять учетными записями. Разработка ведется на Java (Spring Boot) с использованием базы данных PostgreSQL. API предоставляет REST-интерфейсы для взаимодействия с системой.

## Основной функционал

### 1. Разработка REST API
1.1 Создать и запустить локально REST-сервис на Spring Boot.
1.2 Добавить базовые эндпоинты:
    - Получение списка отелей с фильтрацией по дате, городу и категории.
    - Получение информации о конкретном отеле по ID.
1.3 Настроить CheckStyle и исправить стилистические ошибки.
1.4 Возвращать данные в формате JSON.

### 2. Подключение базы данных и реализация CRUD-операций
2.1 Подключить базу данных PostgreSQL.
2.2 Реализовать основные сущности:
    - Пользователь
    - Отель
    - Номер
    - Бронирование
2.3 Настроить связи между сущностями:
    - Один ко многим (@OneToMany) между Отель и Номер.
    - Многие ко многим (@ManyToMany) между Пользователь и Бронирование.
- Реализовать CRUD-операции для всех сущностей.

### 3. Фильтрация и кеширование данных
- Добавить GET-запрос с параметрами, использующий кастомный запрос (@Query) для фильтрации бронирований по имени пользователя.
- Реализовать кеширование запросов в in-memory (например, HashMap) для ускорения доступа к часто запрашиваемым данным.
- Использовать JPQL и native query для оптимизированных запросов в БД.

### 4. Обработка ошибок и логирование
- Реализовать обработку ошибок валидации входных данных (400) с использованием @ControllerAdvice и кастомных исключений.
- Добавить логирование всех действий и ошибок в файл .log.
- Реализовать эндпоинт для получения лог-файла за указанную дату.
- Подключить Swagger и документировать API.

### 5. Дополнительные функции API
- Добавить POST-метод для массового бронирования номеров с использованием Stream API и лямбда-выражений.
- Покрыть основные бизнес-операции unit-тестами (покрытие >80%).

### 6. Асинхронность и нагрузочное тестирование
- Реализовать асинхронный вызов сервиса для создания лог-файла с возвращением ID запроса.
- Добавить эндпоинты для получения статуса по ID и загрузки файла.
- Разработать сервис учета посещений API (подсчет запросов к определенным URL с синхронизированным доступом).
- Провести нагрузочное тестирование с использованием JMeter/Postman.

### 7. Разработка UI
- Создать веб-интерфейс для отображения списка отелей и их номеров (React или Angular).
- Реализовать интерфейс управления бронированиями (создание, удаление, обновление данных).
- Использовать UI-библиотеки: Ant Design, Material UI, Bootstrap.

### 8. Деплой и CI/CD
- Запустить приложение в Docker.
- Разместить сервис на бесплатном хостинге (Render, Railway, Fly.io).
- Настроить CI/CD на GitHub для автоматической сборки и деплоя.

## Технологический стек
- **Backend**: Java, Spring Boot, Spring Data JPA, Spring Security
- **Database**: PostgreSQL
- **Frontend**: React/Angular
- **Build Tools**: Maven
- **Testing**: JUnit, Mockito
- **DevOps**: Docker, GitHub Actions, Swagger
